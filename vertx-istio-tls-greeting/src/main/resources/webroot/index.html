Istio Mutual TLS Mission - Eclipse Vert.x

This mission showcases Istio security concepts whereby access to services is controlled by the platform rather than independently by constituent applications.

This application accesses two services: the greeting service and the name service. The greeting service relies on the name service to retrieve the name. So the name service can be accessed either directly (from this page) or from the greeting service. By default, everybody has access to this service. So, when you click on the invokebuttons, successful responses are displayed.

Once you have invoked the services, we can block the access to the name service if they don't come from the greeting service. So, the greeting service will still work fine, while the name service access from this page will not be authorized. So block the access, execute:

istioctl create -f kubernetes/istio-rule-require-service-account.yaml -n myproject

To remove the access control, execute:

istioctl delete -f kubernetes/istio-rule-require-service-account.yaml -n myproject
